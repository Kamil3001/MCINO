What is a Duplicated Code smell?

When a program contains two sections of code which are identical or extremely similar.

How to treat?

  • Use an Extract Method and place calls for the new method in both places if the same piece of code is found in different methods in the class.
  • If the same code is found in two subclasses of the same level:
    • We can use an Extract Method for both classes. It may be useful to also follow this up using a Pull Up Field.
    • Use Form Template Method if the duplicate code is similar but not completely identical.
    • If two methods have the same function but use two different logic, select the best algorithm and apply Substitute.
  • If duplicate code is found in two different classes:
    • Use Extract Superclass in order to create a single superclass for these classes that maintains all the previous functionality, if the classes arenâ€™t part of a hierarchy,.
    • Use Extract Class in one class and use the new component in the other if its difficult to create a superclass.
  • If a large number of conditional expressions are present and perform the same code (differing only in their conditions), merge these operators into a single condition using Consolidate Conditional Expression and use Extract Method to place the condition in a separate method with an easy-to-understand name.
  • If the same code is performed in all branches of a conditional expression: place the identical code outside of the condition tree by using Consolidate Duplicate Conditional Fragments.

Why bother fixing this smell?

  • Makes code easier to simplify and cheaper to support .
  • Merging duplicate code simplifies the structure of your code and makes it shorter.
